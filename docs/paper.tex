% ─── Abstract ─────────────────────────────────────────────────────────────────
\begin{abstract}
\noindent
We present \textbf{Israel Shield} (\heb{Magen Israel}), a browser-based,
real-time security intelligence dashboard that aggregates, analyzes, and
visualizes over seven years of rocket, missile, drone, and terrorist-infiltration
alerts across Israel. The system integrates a live feed from the IDF Home Front
Command with a curated historical dataset of 10\,000+ records spanning 2019--2026,
processing them through a suite of statistical and algorithmic models. Key
contributions include: (1) a \emph{Spatio-Temporal UAV Route Reconstruction}
algorithm that chains drone alert sequences into probable flight trajectories;
(2) a \emph{Safe Route Departure Advisor} that scores every departure hour using a
sliding-window aggregation over historical threat corridors along a travel path;
(3) a \emph{Shower Index} (\heb{Madad HaMiklahat}), a Poisson-based daily
safety-window estimator; and (4) a fully client-side, offline-capable Progressive
Web App supporting seven languages and bidirectional text. The system demonstrates
that meaningful security analytics are achievable without proprietary
infrastructure, enabling both civilian and academic use cases.

\smallskip\noindent
\textbf{Keywords:} security analytics, threat visualization, spatio-temporal
analysis, UAV tracking, safe route planning, business intelligence, homeland
security, Israel.
\end{abstract}

\section{Introduction}
\label{sec:introduction}
% ═════════════════════════════════════════════════════════════════════════════

Since 2001, Israel has faced persistent rocket fire from Gaza and Lebanon. From
October~2023, an unprecedented multi-front escalation added ballistic missiles,
cruise missiles, and large-scale UAV swarms from Iran, Yemen, Iraq, and Syria
simultaneously. The IDF Home Front Command (\heb{Pikud HaOref}) operates a public
alert system that has generated a historically unique dataset: a timestamped,
city-level record of every incoming-threat warning issued to the Israeli civilian
population.

This dataset presents a rich opportunity for \emph{threat pattern analysis} ---
the systematic study of \emph{when}, \emph{where}, and \emph{how} security
incidents occur at population scale. Despite its public availability, no accessible
multi-functional analytical tool combined live feeds, historical analysis,
geographic visualization, probabilistic safety estimation, and navigation planning
in a single interface.

\textbf{Israel Shield} fills this gap. Built as a Business Intelligence project at
Ariel University, its key distinguishing features are:

\begin{enumerate}
  \item \textbf{UAV Route Reconstruction} --- a novel spatio-temporal chaining
        algorithm reconstructing probable drone flight trajectories from ordered
        city-alert sequences.
  \item \textbf{Safe Route Planner} --- a departure time optimizer scoring each
        hour by historical threat intensity along a proposed travel corridor.
  \item \textbf{Shower Index} --- a Poisson-based daily safety window estimator.
  \item \textbf{Seven temporal resolutions} of alert frequency analysis
        (year down to 1-minute resolution).
  \item \textbf{Full offline capability} via IndexedDB caching and PWA support.
  \item \textbf{Seven languages} with bidirectional RTL/LTR text support.
\end{enumerate}

\section{Background and Related Work}
\label{sec:background}
% ═════════════════════════════════════════════════════════════════════════════

\subsection{Open Security Data in Israel}
The Home Front Command's ``Red Alert'' (\heb{Tzeva Adom}) system is one of the
world's most densely documented civilian warning datasets, timestamped to the
second at city level since 2001~\cite{harpaz2023}. Academic use has been limited;
most analyses have been journalistic or statistical in nature.

\subsection{Threat Visualization Platforms}
Existing tools (LiveUA Map, ACLED) map macro-level conflict without city-level
temporal granularity. None integrate the IDF alert feed's precision or provide
civilian-facing safety planning features.

\subsection{UAV Trajectory Reconstruction}
Trajectory inference from indirect observations is active in air traffic
management~\cite{olive2019}. Prior work requires ADS-B or radar data. This paper
presents the first method using civilian warning activations as trajectory proxies.

\subsection{Route Safety Planning}
Route safety in conflict zones has been studied in humanitarian
logistics~\cite{tomasini2009}. Most approaches optimize \emph{which road} to take;
our approach optimizes \emph{when} to travel on any road.

\subsection{Poisson Models for Safety Windows}
Poisson processes are standard models for rare event occurrence in
time~\cite{kingman1992}, applied here to compute zero-alert interval
probabilities.

\section{Data Sources and Processing Pipeline}
\label{sec:data}
% ═════════════════════════════════════════════════════════════════════════════

\subsection{Data Sources}

\subsubsection{Historical Alert Dataset}
The primary source is a community-curated CSV maintained by Dr.\ Yuval
Harpaz~\cite{harpaz2023}. Each record contains the fields shown in
Table~\ref{tab:schema}.

\begin{table}[H]\centering
\caption{Alert Record Schema}
\label{tab:schema}
\begin{tabular}{lll}\toprule
\textbf{Field} & \textbf{Type} & \textbf{Description} \\\midrule
\texttt{time}     & String & ISO timestamp (\texttt{2023-10-07 06:29:00}) \\
\texttt{cities}   & String & Hebrew city/region name \\
\texttt{threat}   & String & Numeric code or category string \\
\texttt{category} & String & Alternative threat field \\
\texttt{id}       & String & Optional IDF event batch ID \\\bottomrule
\end{tabular}
\end{table}

\noindent The dataset contains $N\!\approx\!10\,000{+}$ records spanning
2019--2026, with density increasing dramatically from October~7, 2023.

\subsubsection{Live Alert Feed}
Real-time alerts are fetched from \texttt{oref.org.il} via an
\texttt{allorigins.win} CORS proxy at 10-second polling intervals.

\subsubsection{Geographic Coordinates}
A \texttt{baseCoords} dictionary hardcodes coordinates for 450+ Israeli cities,
settlements, and regions. For unknown cities, the OpenStreetMap Nominatim API is
queried with rate-limiting (1~req/s, 20~cities/batch) and results cached in
\texttt{localStorage}.

\section{UAV Route Reconstruction Algorithm}
\label{sec:uav}
% ═════════════════════════════════════════════════════════════════════════════

\subsection{Motivation}
When a hostile UAV (\heb{kli tays oyev}) enters Israeli airspace, the Home Front
Command activates alerts in cities sequentially as it approaches each population
center. The time-ordered sequence of city activations implicitly encodes the flight
trajectory.

\subsection{Problem Formulation}

\begin{definition}[UAV Alert Sequence]
Let $\mathcal{A}=\{a_1,\ldots,a_n\}$ be all ``hostile aircraft intrusion'' alerts,
sorted chronologically. Each $a_i=(t_i,\mathbf{p}_i)$ where $t_i$ is the
timestamp and $\mathbf{p}_i=(\phi_i,\lambda_i)$ are geographic coordinates.
\end{definition}

\begin{definition}[UAV Route]
A route $R=(a_{i_1},\ldots,a_{i_k})$ is a maximal subsequence satisfying for each
consecutive pair:
\[
  t_{i_{j+1}}-t_{i_j}\leq\tau_{\max}=10\text{ min}
  \quad\text{and}\quad
  d_H(\mathbf{p}_{i_j},\mathbf{p}_{i_{j+1}})\leq d_{\max}=20\text{ km}
\]
\end{definition}

A 1-2-1 weighted convolution removes zigzag artifacts:
\begin{equation}
  \hat{\mathbf{p}}_i = \frac{\mathbf{p}_{i-1}+2\mathbf{p}_i+\mathbf{p}_{i+1}}{4}
\end{equation}
with endpoints fixed ($\hat{\mathbf{p}}_0=\mathbf{p}_0$, $\hat{\mathbf{p}}_k=\mathbf{p}_k$).
Two passes are applied.

\subsection{Key Equations}

\textbf{Impact zone:}
$\mathcal{Z}=\{c\in\mathcal{C}\mid\exists\,\mathbf{p}\in\text{path}:d_H(\mathbf{p},\mathbf{p}_c)\leq15\text{ km}\}$

\textbf{Window score:}
\begin{equation}
  S(h,\Delta T)=\sum_{i=0}^{\lfloor\Delta T\rfloor-1}c_{(h+i)\bmod 24}
  +c_{(h+\lfloor\Delta T\rfloor)\bmod 24}\cdot\{\Delta T\}
  \label{eq:window}
\end{equation}

\textbf{Quiet probability:}
$P_{\text{quiet}}(h)=\bigl(1-S(h)/\max_{h'}S(h')\bigr)\times100\%$

\section{Shower Index}
\label{sec:shower}
% ═════════════════════════════════════════════════════════════════════════════

\begin{enumerate}
  \item Bucket alerts into 48 half-hour slots:
        $b(a)=\lfloor(60\cdot\text{hour}+\text{min})/30\rfloor$
  \item Weight: $w_i=3c_i+c_{(i-1)\bmod48}+c_{(i+1)\bmod48}$
  \item Find optimal slot, preferring waking hours (07:00--22:00):
        $i^*=\arg\min_{i\in[14,44]}w_i$
  \item Poisson probability: $\lambda=c_{i^*}/|\mathcal{D}|$,\;
        $P(\text{quiet})=e^{-\lambda}$
\end{enumerate}

\begin{proposition}
Under a homogeneous Poisson process with rate $\lambda$ per 30-minute window per
day, $P(\text{quiet})=e^{-\lambda}$ is the probability of observing zero alerts in
a randomly chosen instance of the optimal slot.
\end{proposition}

\section{Implementation Notes}

\subsection{React Anti-Pattern: Inner Function Components}

Defining React components \emph{inside} a parent render function creates a new
component-type identity on every render, causing React to unmount and remount the
component --- resetting all internal state, including keyboard focus:

\begin{lstlisting}[style=tscode, language=JavaScript,
  caption={Incorrect: inner function component loses focus after every keystroke}]
const SafeRouteModal = () => {
  const [city, setCity] = useState('');
  // New type identity on every render -> unmount/remount!
  const InputForm = () => (
    <input value={city} onChange={e => setCity(e.target.value)} />
  );
  return <InputForm />;  // focus lost after each character
};
\end{lstlisting}

\begin{lstlisting}[style=tscode, language=JavaScript,
  caption={Correct: JSX variable; stable identity; focus retained}]
const SafeRouteModal = () => {
  const [city, setCity] = useState('');
  // JSX value, not a new type -> no unmount
  const inputFormJsx = (
    <input value={city} onChange={e => setCity(e.target.value)} />
  );
  return <div>{inputFormJsx}</div>;
};
\end{lstlisting}

\subsection{ECharts Sizing in Flex Containers}

ECharts reads \texttt{clientHeight} at initialization. In CSS flexbox with
\texttt{flex:1} / \texttt{min-height:0}, layout may not be finalized at that
moment. A \texttt{ResizeObserver} solves this reliably:

\begin{lstlisting}[style=tscode, language=JavaScript,
  caption={ResizeObserver for reliable ECharts height}]
const [chartHeight, setChartHeight] = useState(0);
const chartBoxRef = useRef(null);

useEffect(() => {
  const el = chartBoxRef.current;
  if (!el) return;
  const ro = new ResizeObserver(([entry]) => {
    const h = entry.contentRect.height;
    if (h > 0) setChartHeight(h);
  });
  ro.observe(el);
  return () => ro.disconnect();
}, [safeRouteData]);

// JSX:
// <div ref={chartBoxRef} style={{ flex:1, minHeight:0 }}>
//   <ReactECharts style={{ height: chartHeight - 16 }} />
// </div>
\end{lstlisting}

For the documented Houthi UAV attack of October~19, 2023, the algorithm
reconstructed a northward trajectory from the Gulf of Aqaba through the Negev,
consistent with publicly reported flight paths. For the Iranian UAV swarm of
April~13--14, 2024, multiple simultaneous east-to-west chains were reconstructed,
matching the reported approach vector.

Back-testing the Shower Index over October--December 2023 recommends early
morning (03:00--06:00), consistent with the predominantly daytime rocket fire
pattern from Gaza.

For Tel Aviv -- Eilat ($\approx$350\,km, $\sim$4\,h), the Safe Route Planner
identifies 00:00--05:00 as historically safest, consistent with daytime Gaza
rocket patterns and overnight Houthi launches.

\section{Conclusion}
\label{sec:conclusion}
% ═════════════════════════════════════════════════════════════════════════════

We have presented Israel Shield, a browser-based security intelligence dashboard
that transforms a decade of public alert data into actionable civilian safety
intelligence. The system's three core algorithmic contributions --- spatio-temporal
UAV route reconstruction, sliding-window departure time optimization, and the
Poisson-based Shower Index --- demonstrate that meaningful security analytics are
achievable from passively collected public data without specialized sensors or
proprietary feeds.

The application's fully client-side architecture, seven-language support, offline
capability, and PWA installation make it accessible during high-stress security
events when network reliability cannot be assumed. The codebase also illustrates
engineering lessons of broad applicability: the hazards of defining React
components inside render functions, the importance of explicit pixel sizing for
canvas-based charting libraries in flex layouts, and the value of conservative API
fallback strategies.

\section{Window Score Implementation}
\label{app:window}

\begin{lstlisting}[style=tscode, language=JavaScript,
  caption={getWindowScore: scores a departure hour over the full travel window}]
function getWindowScore(
  hourCounts: number[],  // 24-element: alerts per hour
  startH:     number,    // departure hour 0-23
  durationH:  number     // trip duration in hours (fractional ok)
): number {
  if (durationH <= 0) return hourCounts[startH % 24];
  let score = 0;
  const fullHours       = Math.floor(durationH);
  const partialFraction = durationH - fullHours;
  for (let i = 0; i < fullHours; i++) {
    score += hourCounts[(startH + i) % 24];
  }
  if (partialFraction > 0.001) {
    score += hourCounts[(startH + fullHours) % 24] * partialFraction;
  }
  return score;  // lower = safer departure
}
\end{lstlisting}
